⚠️ CASPER BREACH DETECTED!

Security Alert

A decoy passkey was used during login. This indicates that:

  • Encrypted passkey data may have been stolen from cloud storage
  • An attacker attempted to decrypt using a wrong PIN/decoy secret
  • The attacker generated a fake passkey using a decoy secret

How CASPER Detected This:

  1. During registration, RP stored real passkey (V) and decoy passkeys (V')
  2. Attacker stole encrypted data from PMS but didn't know the correct PIN
  3. Attacker tried wrong PIN → selected decoy secret → generated fake passkey
  4. Login attempt used fake passkey public key → key ∈ V' → BREACH DETECTED ✅

Recommended Actions:

  • Revoke the compromised passkey
  • Generate a new passkey with new detection secrets
  • Investigate the security breach
  • Notify the user about the security incident
